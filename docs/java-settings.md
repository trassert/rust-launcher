## Настройки Java и JVM-аргументы

Раздел **«Настройки → Игра → Java»** позволяет управлять тем, какую Java и какие JVM-аргументы использует игра.

- **Путь к Java** — при необходимости можно указать собственный `javaw.exe/java`. Если поле пустое, лаунчер использует встроенный runtime Mojang.
- **Память JVM (MIN/MAX)** — задаёт флаги `-Xms` и `-Xmx`. Значения указываются в мегабайтах или гигабайтах (например, `1024M`, `2G`, `4G`). Лаунчер контролирует, чтобы `Xms ≤ Xmx` и чтобы `Xmx` не превышал объём ОЗУ минус 1–2 ГБ.
- **Пользовательские JVM-аргументы** — произвольные флаги JVM, вводимые одной строкой или построчно. Поддерживаются плейсхолдеры:
  - `${classpath}` — итоговый classpath игры;
  - `${natives}` — путь к папке нативных библиотек;
  - `${gameDir}` — рабочая папка игры;
  - `${assetsDir}` — папка с ассетами;
  - `${version}` — идентификатор версии Minecraft.

### Пресеты

Доступно несколько рекомендованных пресетов:

- **Баланс** — умеренное потребление памяти и плавная работа на большинстве систем.
- **Максимальная производительность** — повышенные значения памяти и более агрессивная настройка GC; подходит для мощных ПК.
- **Низкая память** — минимально возможные безопасные значения для систем с небольшим объёмом ОЗУ.

Каждый пресет задаёт Xms/Xmx и базовый набор флагов `-XX`, которые затем можно дополнительно изменить вручную.

### Безопасность

Лаунчер:

- не пропускает заведомо опасные флаги (`-agentlib`, `-agentpath`, `-Xrun` и т.п.);
- помечает экспериментальные флаги (`-XX:+AggressiveOpts`, `-XX:+UnlockExperimentalVMOptions`) предупреждениями;
- ограничивает `Xmx` так, чтобы не забирать всю доступную память системы.

Кнопка **«Проверить»** выполняет короткий запуск `java -XshowSettings:vm -version` с указанными аргументами и показывает ошибки/предупреждения и вывод Java. Это позволяет безопасно проверить конфигурацию до реального запуска игры.
